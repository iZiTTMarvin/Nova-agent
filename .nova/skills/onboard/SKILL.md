# Nova Agent 入门向导

你正在执行 **onboard** 技能，帮助用户完成首次配置与熟悉工作区。

## 当前环境

- 工作区路径：<%= workspacePath %>
- 当前分支：<%= gitBranch %>

## 执行步骤

1. **问候并确认目标**  
   用简短中文问候用户，询问他们今天想完成什么（探索代码、修 bug、加功能等）。

2. **扫描工作区结构**  
   使用 `ls` / `read` 了解项目根目录布局（README、package.json、src 等），用 3–5 句话概括项目类型与技术栈。

3. **检查 Nova 配置**  
   - 是否存在 `AGENTS.md`、`CLAUDE.md` 或 `.nova/rules/`  
   - 是否存在 `.nova/skills/` 下的项目级技能  
   若缺失，建议用户可在「设置 → 规则 / 技能」中创建。

4. **介绍 Slash 技能**  
   说明用户可在输入框输入 `/技能名` 调用技能，例如 `/skill-creator` 创建技能、`/skill-add` 安装技能。

5. **收尾**  
   给出 2–3 条可立即尝试的下一步（例如「让我帮你读 README」「运行测试」「创建第一个 skill」）。

## 约束

- 不要未经用户同意修改文件；本向导以**只读探索**为主。
- 回复使用简体中文，语气友好、简洁。
- 若工作区为空或非代码项目，如实说明并引导用户选择合适目录。
